### 2. **PDF Structure** (Matching Reference Format)

**Header Section:**
- Title: "115-KOST CASE ANOMALI BOOSTER COMPRESSOR B CPP DONGGI"
- Model/System: J.BEC-UAD
- Timestamp: (Anomaly datetime)
- Equipment: BOOSTER COMPRESSOR B CPP DONGGI
- Location: CPP Donggi

**Main RCA Table Columns:**

| Column | Description |
|--------|-------------|
| SYMPTOM | Division name (INSTRUMENT, PROCESS, MECHANICAL, ELECTRICAL, RELATIONAL) |
| SUB | Division code (305, 390, 440, 525, 225) |
| PROB | Scenario number within division (0, 1, 2, ...) |
| RCA | Detailed root cause analysis description |
| ACTIONS | Recommended corrective actions |
| Flow Rate | YES/NO if this sensor is affected |
| Suction Press | YES/NO if this sensor is affected |
| Discharge Press | YES/NO if this sensor is affected |
| Suction Temp | YES/NO if this sensor is affected |
| Discharge Temp | YES/NO if this sensor is affected |
| SYMPTOM | Detailed symptom description |
| CODE | Symptom code (e.g., S.S.F.A, D.S.P.A, etc.) |

**Summary Row:**
- Anomaly statistics
- Divisions impacted
- Gas loss estimate

**Footer:**
- Generated by GUARD
- Generation timestamp
- Number of scenarios evaluated

- INST-0: Flow Transmitter Malfunction
- INST-1: Suction Pressure Transmitter Dysfunction
- PROC-2: Fouler Gas Preparation Check
- MECH-4: Suction Strainer Check

**Scenarios are ranked by match score:**
- Scenarios addressing more abnormal sensors appear first
- Maximum 8-10 most relevant scenarios included
- Prevents information overload
